1. Reconeix la utilitat de les aplicacions d’ofimàtica web
Explica per què GitHub es pot considerar una aplicació web de treball col·laboratiu.
GitHub és una aplicació web perquè funciona a través d’un navegador i permet a usuaris treballar junts en un mateix projecte.
Quins avantatges té utilitzar GitHub en lloc de guardar els fitxers d’un projecte només en un ordinador local?
Accés des de qualsevol lloc, treball en equip, historial de versions

Com pot ajudar GitHub a gestionar versions d’un projecte web o d’una aplicació?
GitHub permet guardar totes les modificacions del projecte, veure qui ha fet cada canvi
En quins tipus de projectes és especialment útil utilitzar Git i GitHub?
Projectes web, aplicacions mòbils, projectes col·laboratius.

Quines diferències hi ha entre un sistema de control de versions com Git i un sistema tradicional d’emmagatzematge de fitxers?
Git guarda un historial complet de canvis, Permet treballar en branques
facilita la col·laboració, els sistemes tradicionals només guarden fitxers sense control avançat de versions



2. Classifica segons la funcionalitat i prestacions específiques
Explica quina és la diferència entre Git i GitHub.
Git és una eina de control de versions
GitHub és una plataforma web que utilitza Git

Quines funcionalitats ofereix GitHub a més del control de versions del codi?
Col·laboració entre usuaris, revisió de codi, gestió de projectes, pull requests


Esbrina i defineix les següents funcionalitats de GitHub segons el seu ús:
repositoris: Són espais on es guarda el projecte, amb tots els fitxers i el seu historial de canvis.
issues: Són eines per gestionar tasques, errors o suggeriments dins d’un projecte.
pull requests: Són sol·licituds per incorporar canvis d’una branca a una altra, permetent revisar-los abans d’acceptar-los.
GitHub Pages: És una funcionalitat que permet publicar pàgines web directament des d’un repositori de GitHub.
Quina funció tenen les branques (branches) en Git?
Permeten treballar en diferents versions del projecte de manera paral·lela sense afectar la versió principal.

Per a què serveixen les pull requests en un projecte col·laboratiu?
Serveixen per revisar, comentar i aprovar canvis abans d’integrar-los al projecte principal.
3. Gestiona els comptes d’usuari
Què és un compte d’usuari a GitHub i per què és necessari per treballar amb repositoris remots? És un perfil personal que permet accedir a GitHub, crear repositoris i col·laborar en projectes remots.
Quina diferència hi ha entre un repositori públic i un privat a GitHub?
Públic: tothom pot veure el contingut
Privat: només els usuaris autoritzats poden accedir
Com es pot afegir un col·laborador a un repositori de GitHub?
Des de la configuració del repositori, afegint l’usuari a la secció de col·laboradors i assignant-li permisos.
Quins rols o permisos pot tenir un usuari dins d’un repositori?
Administrador
Escriure
Lectura 

Per què és important gestionar correctament els permisos dels usuaris en un projecte col·laboratiu?
Per protegir el projecte, evitar errors, controlar qui pot modificar el codi i garantir una bona organització del treball en equip.


4. Aplica criteris de seguretat en l’accés dels usuaris

    16. Quins riscos pot tenir compartir un repositori amb massa permisos per a tots els usuaris?
        Pot provocar que qualsevol usuari pugui modificar o eliminar fitxers importants del projecte, generant errors o perdúa d'informació.

    17. Per què és recomanable utilitzar autenticació segura (com tokens o SSH) per accedir a GitHub?
        Perquè dóna més seguretat que una contrasenya i evitar accessos no autoritzats al compte.

    18. Què passaria si es publiquen contrasenyes o claus privades dins d’un repositori?
        Altres persones podrien accedir als serveis privats o comptes, serien problems de seguretat greus.

    19. Quines bones pràctiques de seguretat s’han de seguir quan es treballa amb GitHub?
        No compartir dades sensibles, es millor utilitzar autenticació segura, control de permisos i revisar els canvis abans de pujar-los.
    
    20. Per què és important controlar qui pot fer push directament a la branca principal (main)?
        Per evitar liar-la a lo grande, mantenir el projecte estable, ja que només persones autoritzades haurien de modificar la versió final.


5. Utilitza les aplicacions de forma cooperativa
    
    21. Explica com poden treballar diverses persones al mateix projecte utilitzant GitHub.
        Cada persona treballa a la seva pròpia branca i després es fusionen els canvis, per a no interferir amb el treball de l'altre.
    
    22. Per què és útil treballar amb branques en lloc de modificar directament la branca principal?
        Per a fer proves sense afectar la versió principal del projecte.


6. Elabora documentació relativa a l’ús i la gestió de les aplicacions
    
    23. Per què és important incloure un fitxer README.md en un repositori de GitHub?
        Perquè explica el funcionament del projecte i com utilitzar-lo.

    24. Quins avantatges té utilitzar Markdown per escriure documentació a GitHub?
        La info és clara i fàcil de organitzar.
    
    25. Com pot ajudar la documentació a altres desenvolupadors que volen utilitzar o modificar el projecte?
        Ajuda a entendre el contingut del projecte i fer canvis correctament.
